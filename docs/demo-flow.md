# Demo Flow

## 15-Minute Timeline

| Time | Act | What to do |
|---|---|---|
| 0:00 | Opening | Open PR in browser, show customized repo |
| 2:00 | Code Review | Show Droid's PR review comments (SQL injection, PII leak) |
| 5:00 | Missions | Paste feature request, watch Droid implement across 4+ files |
| 9:00 | Ticket Enrichment | Open vague Linear ticket, show spec mode |
| 12:00 | Platform | Show MCP, BYOK, custom droids, or other requested features |
| 14:00 | Wrap | Key takeaway, Q&A |

## Quick Version (7-8 minutes)

| Time | Act | Focus |
|---|---|---|
| 0:00 | Opening + Review | Show 1-2 review comments |
| 2:00 | Mission | Feature implementation |
| 5:00 | Ticket | Spec mode demo |
| 7:00 | Wrap | Differentiators |

## Persona Adaptations

- **VP/Director**: More ROI, less live coding. Show governance.
- **Staff/Principal**: Deep code review reasoning. Architecture focus.
- **IC Developer**: Hands-on, speed-focused. VS Code integration.
- **Eng Manager**: Balanced. Velocity metrics + code quality.
- **Product Manager**: Linear workflow. Skip raw code.
